Using if and switch Statements

  • ifs sem formatação

    if(exam.done())
if(exam.getScore() < .95)
System.out.println("Try again.");
else System.out.println("Java Master!");

if(exam.done()) {
    if(exam.getScore() < .95) {
        System.out.println("Try again.");
    }
}
else {
    System.out.println("Java Master!");
}

  • a && b | c && d = a && (b | (c && d))

  • switch

    switch(expression) {
    case constant1: code;
    case constant2: code;
    default: code;
}

// expression -> char, byte, short, int, enum, String
// constant -> literal ou final variable INICIALIZADA NA DECLARAÇÃO

final int a = 1;
final int b;
b = 2;
int x = 0;
switch(x) {
    case a:
    case b:
}

// constant -> deve pertencer ao range do tipo da expression
// OK
byte a = 3;
byte b = 2;
switch(a + b) {
    case 128:
}
// COMPILATION ERROR
byte a = 5;
switch(a) {
    case 128:

  • "It's also illegal to have more than one case label using the same value."

  • "case constants are evaluated from the top down"

Creating Loop Constructs

  • for(/ initialization /;;)
    • se for declarar, só pode declarar variáveis de um tipo; ex: for(int a = 10, b = 20;;)
    • se não for declarar, pode inicializar de diversos tipos
  • for(;/ condition /;)
    • se ausente, true; ex: for(;;) {} // loop infinito
  • for(;;/ iteration /)
    • pode ser qualquer statement não bloco(na verdade, não sei delimitar o que pode, mas bloco não pode), separados por vírgula
  • for(int i = 10, j = 20; i < j; i++, System.out.println(i), j+=2) {}

  • "when the continue statement is hit, the iteration expression still runs!"

    fora: for(int x = 0; x < 1; x++) {
    for(;; System.out.println("dentro")) {
        continue fora;
    }
}
// resultado -> nada é impresso, o continue ativa o código de iteration do loop que ele continua

  • "Labeled continue and break statements must be inside the loop that has the same label name, otherwise, the code will not compile."

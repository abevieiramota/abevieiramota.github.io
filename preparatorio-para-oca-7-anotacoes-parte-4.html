In Java.

Literal, Assignments, and Variables

  • "A primitive literal is merely a souce code representation of the primitive data types"

  • "As of Java 7, numeric literals can be declared using underscore characters(_)"

    • não pode no:
      • início
      • fim
      • antes ou depois de vírgula
      • antes, dentro ou depois de indicador de base

  • binary literals: starts with 0b or 0B

  • octal literals: starts with 0
  • hexadecimal literals: starts with 0x or 0X
    • 0XCAFE = 0xcafe
  • by default, a integer literal is int, but you can place suffix L or l to turn it into long

  • floating-point literals are defined as double by default

    float a = 12.3; // COMPILATION ERROR - possible loss of precision
float a = 12.3F; // OK
float a = 12.F; // OK
float a = .3F; // OK

  • character literals

    char a = 'a';
char b = '\u004e';

  • caracteres são 16 bit unsigned, podem receber numerais que pertençam ao seu range ou via cast

    char a = 982;
char b = (char) 70000; // int value 4464

Assignment Operators

  • toda operação com numéricos inteiros de tipo menor ou igual a int resultam em int

    byte a = 1;
byte b = 0;
byte c = a + b; // COMPILATION ERROR

byte c = 1 + 0; // OK
final byte a = 1;
final byte b = 0;
byte c = a + b; // OK

  • "Typically, an implicit cast happens when you're doing a widening conversion - in other words, putting a smaller thing(say, a byte) into a bigger container(say, a long)"

  • "The large-value-into-small-container conversion is referred to as narrowing and requires an explicit cast"

    int x2 = 20;
short y = x2; // COMPILATION ERROR

final x1 = 20;
short y = x1; // OK

long a = 10;
float b = a;

  • cast de número decimal para inteiro -> perde a parte decimal

  • "when you narrow a primitive, Java simply truncates the higher-order bits that won't fit"
  • "The compound assingment operator += lets you add to the value of b, without putting in an explicit cast. In fact, +=, -=, *=, and /= will all put in an implicit cast."

Scope

  • escopo de variáveis
    • static: criado quando a classe é carregada e mantido enquanto a classe estiver carregada
    • instance: criado quando uma instância é criada e mantido enquanto a instância não for removida
    • local: mantido enquanto o método onde está declarada estiver na stack
    • block: mantido enquanto o bloco estiver executando

Variable Initialization

  • "instance variables are initialized to a default value each time a new instance is created"
  • char default value = '\u0000'
  • array não inicializado = null
  • array inicializado = todos slots com valores default
  • "local variables, including primitives, always, always, always must be initialized before you attempt to use them"
    class A {
    public static void main(String[] args) {
        String nome;
        if(nome == null) { // COMPILATION ERROR
            System.out.println("oi");
        }
    }
}

Passing Variables into Methods

  • "Java is actually pass-by-value for all variables running within a single JVM"
  • Shadowing -> apenas se uma for de escopo static ou instance e a outra local ou block

Garbage Collection

  • "an object is eligible for garbage collection when no live thread can access it"

Java Operators

  • "if a floating-point number is compared with an integer and the values are the same, the == operator usually returns true as expected"

  • "The instanceof operator is used for object reference variables only"

    class A{}
class B{
    public static void main(String[] args) {
        B b = new B();
        if(b instanceof A) {
            //  COMPILATION ERRO -> B não pertence é subclasse de A
        }
    }
}

  • null instanceof X = false

  • ((A) null) instanceof A = false
  • new B[10] instanceof A[] = true, se B extends A
  • += com String
    String a = "oi";
B b = new B();
a += b; // OK - chama toString

Using String and StringBuilder

  • String é final
  • String s = new String("asaas"); -> cria 2 objetos
  • equals, equalsIgnoreCase (null) = false
  • "0123456".substring(1, 3) = "12"
  • new StringBuilder().append("as").reverse().insert(2, "ias").toString();

Using Arrays

  • arrays are objects
  • acesso a posição fora dos limites do array -> RuntimeException

  • inicialização

    int[] arr = {1, 2, 3};
int[] arr2;
arr2 = {1, 2, 3}; // COMPILATION ERROR
arr2 = new int[] {1, 2, 3}; // OK

  • "Primitive array can accept any value that can be promoted implicitly to the declared type of the array"

    int[] arr = new int[3];
arr[0] = (byte) 4;
arr[1] = 'c';
arr[2] = (short) 3;

char[] charr = new char[3];
int[] arr = charr; // COMPILATION ERROR
B[] bs = new B[2];
A[] as = bs; // OK
bs = new B[1000]; // OK
